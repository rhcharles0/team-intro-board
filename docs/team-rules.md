# 팀 규칙
1) main 에 직접 push 금지
2) 모든 작업은 feature branch 에서
3) main 반영은 PR 통해서만
4) PR 머지 전 최소 1명 리뷰

# 컨벤션 규칙
[#이슈번호] feat: 새로운 기능 추가
[#이슈번호] fix: 버그 수정
[#이슈번호] style: html, css 등 디자인
[#이슈번호] docs: 문서 수정
[#이슈번호] refactor : 코드 리펙토링
[#이슈번호] test : 테스트 코드, 리펙토링 테스트 코드 추가
[#이슈번호] chore: 빌드 업무 수정, 패키지 매니저 수정
